# Image

### A container for **responsive images**

[Bulma Documentation](https://bulma.io/documentation/elements/image/)

Supported Versions: `0.9.1`, `0.9.0`

## Default

Because images can take a few seconds to load (or not at all), use the `<Image>` container to specify a **precisely sized** container so that your layout isn't broken because of image loading or image errors.

The `<Image.Content>` subcomponent is provided as an wrapper for the `<img>` tag.

<Canvas>
  <Story name="Default">
    <Image size="128x128">
      <Image.Content src="https://bulma.io/images/placeholders/128x128.png" />
    </Image>
  </Story>
</Canvas>

## Image Ratios

If you don't know the exact dimensions but know the **ratio** instead, you can use one of the **16 ratio modifiers**. Reference the `ratio` prop to see all the values.

<Canvas>
  <Story name="Ratios">
    <Block style={{ maxWidth: '240px' }}>
      <h1>Square</h1>
      <Image ratio="square">
        <Image.Content src="https://bulma.io/images/placeholders/480x480.png" />
      </Image>
    </Block>
    <Block style={{ maxWidth: '240px' }}>
      <h1>4 by 3</h1>
      <Image ratio="4by3">
        <Image.Content src="https://bulma.io/images/placeholders/640x480.png" />
      </Image>
    </Block>
    <Block style={{ maxWidth: '240px' }}>
      <h1>16 by 9</h1>
      <Image ratio="16by9">
        <Image.Content src="https://bulma.io/images/placeholders/640x360.png" />
      </Image>
    </Block>
  </Story>
</Canvas>

The `<Image>` container will usually take up the whole width while maintaining the perfect ratio.
If it doesn't, you can force it by adding the `fullwidth` prop.
