# Select

### The browser built-in **select dropdown**, styled accordingly

[Bulma Documentation](https://bulma.io/documentation/form/select/)

Supported Versions: `0.9.1`, `0.9.0`

## Default

The `<Select>` is a container for the [HTML select](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/select), which gives the styling more flexibility and support for icons.

The `<Select.Content>` and `<Select.Option>` subcomponents are provided as wrapper for the native `<select>` and `<option>` HTML elements.

<Canvas>
  <Story name="Default">
    <Select>
      <Select.Content>
        <Select.Option>Select dropdown</Select.Option>
        <Select.Option>With options</Select.Option>
      </Select.Content>
    </Select>
  </Story>
</Canvas>

## Icons

You can append the `iconsLeft` prop on a control with the `align="left"` prop on the icon.

The size of the **select** will define the size of the `<Icon>` container.

<Canvas>
  <Story name="Icons">
    <Field>
      <Control iconsLeft>
        <Select>
          <Select.Content>
            <Select.Option selected>Country</Select.Option>
            <Select.Option>United States</Select.Option>
            <Select.Option>Canada</Select.Option>
            <Select.Option>Mexico</Select.Option>
          </Select.Content>
        </Select>
        <Icon size="small" align="left">
          <FontAwesomeIcon icon={faGlobe}/>
        </Icon>
      </Control>
    </Field>
  </Story>
</Canvas>

If the control contains an icon, Bulma will make sure the icon remains **centered**, no matter the size of the input *or* of the icon.
