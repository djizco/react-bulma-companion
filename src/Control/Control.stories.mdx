# Control

### All generic **form controls**, designed for consistency

[Bulma Documentation](https://bulma.io/documentation/form/general/#form-control)

Supported Versions: `0.9.4`, `0.9.3`, `0.9.2`, `0.9.1`, `0.9.0`

## Default

The `<Control>` is a versatile **block** container meant to **enhance single form controls**. Because it has the same height as the element that it wraps, it can **only contain** the following components:

- `<Input>`
- `<Select>`
- `<Button>`
- `<Icon>`

## Form Addons

If you want to **attach** `<Controls>` together, use the `addons` prop on the `<Field>` container:

<Canvas>
  <Story name="Form addons">
    <Field addons>
      <Control>
        <Input placeholder="Text input"/>
      </Control>
      <Control>
        <Button color="info" component="a">Search</Button>
      </Control>
    </Field>
  </Story>
</Canvas>

You can attach inputs, buttons, and dropdowns **only**.

It can be useful to append a **static button**.

<Canvas>
  <Story name="Form addons with static">
    <Field addons>
      <Control>
        <Input type="email" placeholder="Your email"/>
      </Control>
      <Control>
        <Button static>@gmail.com</Button>
      </Control>
    </Field>
  </Story>
</Canvas>

If you want to **group** controls together, use the `grouped` prop on the `<Field>` container.

## Horizontal Form Control

If you want a horizontal form control, use the `horizontal` prop on the `<Field>` container, in which you include:

- `<Field.Label>` for the side label
- `<Field.Body>` for the input/select/textarea container

You can use `grouped` or `addons` for the child elements.
