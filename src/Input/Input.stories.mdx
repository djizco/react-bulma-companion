# Input

### The **text input** and its variations

[Bulma Documentation](https://bulma.io/documentation/form/input/)

## Default

The `<Input>` component is a wrapper for the [HTML input](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/input). The following type attributes are supported:

- `type="text"`
- `type="password"`
- `type="email"`
- `type="tel"`

By default, it will be a text input.

## Readonly and Static

If you add the `readOnly` prop, the input will look similar to a normal one, but is not editable and has no shadow.

<Canvas>
  <Story name="Readonly">
    <Input value="This text is readonly" readOnly />
  </Story>
</Canvas>

If you *also* append the `static` prop, it removes the background, border, shadow, and horizontal padding, while maintaining the **vertical spacing** so you can easily align the input in any context, like a horizontal form.

<Canvas>
  <Story name="Static">
    <React.Fragment>
      <Field horizontal>
        <Field.Label size="normal">
          <Label>From</Label>
        </Field.Label>
        <Field.Body>
          <Field>
            <Control>
              <Input type="email" value="me@example.com" static readOnly />
            </Control>
          </Field>
        </Field.Body>
      </Field>
      <Field horizontal>
        <Field.Label size="normal">
          <Label>To</Label>
        </Field.Label>
        <Field.Body>
          <Field>
            <Control>
              <Input type="email" placeholder="Recipient email" />
            </Control>
          </Field>
        </Field.Body>
      </Field>
    </React.Fragment>
  </Story>
</Canvas>

## Icons

These examples use [React Fontawesome](https://github.com/FortAwesome/react-fontawesome) with Font Awesome icons.

You can append one of 2 modifier **props** on a `<Control>`:

- `iconsRight`
- `iconsLeft`

You also need to add a modifier prop on the `Icon`:

- `<Icon align="left" />` if `<Control iconsLeft />`
- `<Icon align="right" />` if `<Control iconsRight />`

The size of the `<Input>` will define the size of the `<Icon>` container.

<Canvas>
  <Story name="Icons">
    <Field>
      <Control iconsLeft iconsRight>
        <Input type="email" placeholder="Email" />
        <Icon size="small" align="left">
          <FontAwesomeIcon icon={faEnvelope} />
        </Icon>
        <Icon size="small" align="right">
          <FontAwesomeIcon icon={faCheck} />
        </Icon>
      </Control>
    </Field>
  </Story>
</Canvas>

If the `<Control>` contains an `<Icon>`, Bulma will make sure the `<Icon>` remains **centered**, no matter the size of the `<Input>` or of the `<Icon>`.
